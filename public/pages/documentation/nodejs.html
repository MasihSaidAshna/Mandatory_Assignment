Node.js and REST API conventions

General documentation of Node.js and REST API conventions

Node.js is an open-source JavaScript runtime environment. A key feature behind Node.js is its package system, npm, which is used to access innumerous libaries and modules. Node.js has the benefit of server-side rendering, which means the server is often used to generate dynamic HTML content before the page is sent to the client. THanks to its event-driven architecture, Node.js is capable of asynchronous I/O. This means appliations can handle many connections at the same time and respond quickly. It is ideally suited for building REST APIs.

Using Node.js in the terminal

Usage Node.js comes with using your terminal a lot of the time. The table below shows some examples of common commands one would type in the terminal when using Node.js.

Terminal command Description
npm init Initializes/sets up a project with defaults; creates package.json
npm install package_name Installs a specific package
node filename.js Runs the Node.js script from the project directory

REST API conventions

There are 3 conventions when it comes to REST:
  1. HTTP methods (verbs)
  2. Ordering
  3. Naming nouns in plural or singular

HTTP methods

REST APIs use HTTP methods to perform different actions. The most commonly used are GET, POST, PUT, DELETE. GET is used to retrieve data from a resource, POST creates a new resource, PUT updates an existing resource and DELETE deletes a resource.

Ordering

Ordering in API responses. This convention is about maintaining consistent ordering of resources which in turn increases usability and maintainability of te API.

Naming

The naming conventions for resources in REST API follow a pattern of nouns for resources and plural nouns for collections of resources. An example of this could be retrieving a user, where the endpoint would be "/user". When retrieving a collection of users, this endpoint would then be "/users".
